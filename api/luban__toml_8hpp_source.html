29// Per-project toolchain version pins (OQ-2). Maps component name (as it
30// appears in installed.json — "cmake" / "ninja" / "llvm-mingw" / etc.) to
31// a required version string. `luban build` checks pins against the
32// registry on launch and warns on mismatch (it doesn't auto-install or
33// hard-fail by default — that escalation is left to a future flag).
34//
35// Empty map = no pins, no checks. Symmetric with [scaffold].sanitizers
36// being optional.
37//
38// Format in luban.toml:
39//
40// [toolchain]
41// cmake = "4.3.2"
42// ninja = "1.13.2"
43// llvm-mingw = "20260421"
44//
45// Versions are matched as raw strings (no semver range parsing in v1).
46// vcpkg / cmake / ninja release tags are mostly date- or dotted-int form
47// already; range support can come later if a real ask emerges.
48using ToolchainPins = std::map<std::string, std::string>;
49
50struct Config {
51 ProjectSection project;
52 ScaffoldSection scaffold;
53 ToolchainPins toolchain;
54};
