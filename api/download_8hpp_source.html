46 // Chunked HTTP Range download. When `parallel_chunks > 1` AND the server
47 // returns a HEAD response with Content-Length >= chunk_threshold AND
48 // accepts byte ranges, luban issues N concurrent GETs with
49 // `Range: bytes=<lo>-<hi>` headers and stitches the file together.
50 // Primarily a CDN-throttle bypass: a single TCP stream is rate-limited
51 // by the server's per-connection cap; multiple connections aggregate
52 // closer to link ceiling. For LLVM-MinGW (~280 MB) the speedup is
53 // typically 1.5-3x.
54 //
55 // Defaults to 0 (single-stream — identical to v0.1.x behavior). Component
56 // install overrides to 4. Falls back to single-stream cleanly when HEAD
57 // fails, the file is small, or the server doesn't advertise byte ranges.
58 int parallel_chunks = 0;
59 int64_t chunk_threshold = 8 * 1024 * 1024; // 8 MiB
60};
