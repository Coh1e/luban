2// PATH-lookup helper. Single owner of the logic that previously lived as
3// near-identical copies in doctor.cpp and perception.cpp (and a third Win32
4// variant in commands/which_search.cpp).
5//
6// On Windows: SearchPathW with explicit PATHEXT iteration so .cmd / .bat
7// shims are picked up too (SearchPathW alone skips them when a bare name
8// is queried). On POSIX: split $PATH on ':' and test fs::exists.
9//
10// The lookup honours **the current process PATH** — callers that want to
11// search a luban-augmented PATH (e.g. commands/which_search.cpp does this
12// to include toolchain dirs not yet on user PATH) must SetEnvironmentVariable
13// before calling and restore after. This keeps the helper stateless.

23// Resolve `tool` against $PATH (or %PATH%). Returns absolute path on hit,
