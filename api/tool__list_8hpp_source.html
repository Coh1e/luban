2// Canonical tool name lists shared by `luban doctor` and `luban describe --host`
3// (perception). Two layers:
4//
5// kCoreTools — must-be-on-PATH for a working luban-managed dev env. Doctor
6// emits ✗ when any of these is missing; --strict treats it as
7// a CI-gate failure. Add an entry only if its absence breaks
8// a daily-driver flow (build / debug / lint).
9//
10// kEcosystemExtras — informational. Useful neighbours (Node, Python, uv) +
11// luban-specific tools (doxygen for `luban doc`, doctest
12// for unit tests). Probed by perception::snapshot() but
13// not enforced; missing one is never a failure.
14//
15// Perception reports the union (core + extras). Doctor only reports core.
16//
17// Why a shared header: previously doctor.cpp and perception.cpp each owned a
18// constexpr array with the first 9 entries duplicated verbatim. Adding a tool
19// required touching both, which we'd already drifted on once.
