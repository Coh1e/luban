2// Host-capability detection.
3//
4// Sniffs OS / CPU / RAM / available SIMD / known dev tools / XDG env so
5// luban (and consumers like AGENTS.md generation, IDE plugins, AI agents)
6// can give project-specific advice instead of blanket recommendations.
7//
8// Surfaced via `luban describe --host` (machine-readable JSON) and as a
9// short summary in `luban doctor`. Pure read-only — never mutates state.
10//
11// Design notes:
12// * Zero new dependencies — uses Win32 (GetSystemInfo, __cpuid,
13// GlobalMemoryStatusEx) on Windows; /proc and uname on POSIX (M4).
14// * Detection is best-effort. A field that can't be resolved becomes an
15// empty string / -1 / nullopt rather than a hard error — agents can
16// work with partial info.
17// * No network. No registry probing beyond what's already needed by
18// paths.cpp.
19// * GPU / NPU detection is out of scope for v1 (DirectML / WinML pull
20// in COM); see open-questions.md.
